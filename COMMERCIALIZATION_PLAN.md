# Dot Arena 最小商业化改造计划

## 1. 执行目标

这份文档只记录当前仓库里已经落地、正在推进、还未完成的事项，作为 AI agent 的执行基线。它不再讨论方向，只维护状态和下一步动作。

当前目标很明确：

- 把现有可运行原型整理成最小可商业化结构
- 保持单机与联机共用同一套对局核心
- 逐步补齐对局闭环、局外闭环、内容复玩性和基础运营外壳

## 2. 当前已完成

以下内容已经在仓库中形成稳定结果，应视为 `已完成`，后续只做联动补充，不再回退。

### 2.1 完整对局流程

- 对局流程已形成 `Entry -> Matchmaking -> InMatch -> Settlement` 闭环
- 匹配中不再只是文案切换，而是独立流程节点
- 结算态已独立出来，支持 `Play Again` 和 `Return to Lobby`
- 单机和联机共用同一套前台流程

### 2.2 大厅 scene UI 可交互

- 大厅 scene UI 已具备可交互能力
- 任务、商店、设置可直接在大厅面板上操作
- 大厅不再只是只读展示，而是具备局外操作入口
- 这层结构已经可以承接后续正式大厅页面

### 2.3 局外成长持久化

- 局外成长数据已支持按玩家档案持久化
- 已有等级、经验、软货币、任务、商店、战绩、设置等基础数据骨架
- 这层数据已经可以承接更完整的局外成长和结算联动

### 2.4 新增 pickup

- 已新增 `Shield` pickup
- 已新增 `BonusScore` pickup
- 护盾状态已接入对局判定和表现层
- 新 pickup 已同步到客户端展示和提示层

### 2.5 装备影响局内表现

- 本地已装备皮肤会影响对局内玩家颜色
- 局外成长已经开始影响局内可见结果
- 这说明装备与战斗表现之间的链路已经打通

### 2.6 联机大厅主状态

- 联机账号密码登录成功后已进入 `联机大厅`
- 联机匹配取消后会返回 `联机大厅`
- 联机对局结算后可返回 `联机大厅`
- 单机结算返回 `Mode Select`，联机与单机已明确分流

## 6. 推荐执行顺序

这部分是给 AI agent 的执行约束，不建议并行拆散。

1. 先把 `大厅 / 匹配 / 结算` 的产品化收口做稳
2. 先把 `联机大厅首页 / 单机入口 / 内容预览入口` 分层做清楚
3. 再把 `局外成长 / 任务 / 商店 / 设置` 串成统一本地闭环
4. 接着补 `排行榜 / 战绩产品化`
5. 然后补 `内容层`
6. 最后补 `稳定性与运营工具`

## 7. 下一步重点

当前项目已经从可运行原型进入最小商业结构阶段。接下来最重要的不是再开新系统，而是继续把已有模块做成统一产品结构。

下一阶段应优先推进如下：

- 联机大厅首页产品化
- 结算页产品化
- 战绩与排行榜产品化
- 地图机关与主动技能
- 断线重连与正式匹配队列

其中第一优先级不是继续堆底层，而是把 `联机大厅 -> 匹配 -> 对局 -> 结算 -> 回联机大厅` 这条产品主链路做完整。
